UI Testing

User interface automation testing

üéØ Selectors & Locators

  • Use data-testid for stable selectors
  • Avoid CSS classes meant for styling as selectors
  • Prefer roles and accessible names
  • Use page.getByRole() for semantic elements

‚è±Ô∏è Waits & Timeouts

  • Use expect().toBeVisible() instead of sleep
  • Configure reasonable timeouts for slow operations
  • Wait for content loading with waitForLoadState
  • Use waitForSelector for dynamic elements

üì± Responsive & Devices

  • Test on different screen sizes
  • Use viewport configuration
  • Check mobile gestures and events
  • Test touch events separately

üîç Interactions

  • Click, type text step by step
  • Check states before and after actions
  • Test keyboard navigation
  • Use hover() for hover effects

Unit Testing

Testing individual functions and components

üß© Isolation & Mocks

  • Mock external dependencies
  • Use vi.mock() for modules
  • Test one function at a time
  • Avoid testing implementation details

üìã Test Structure

  • Follow Arrange-Act-Assert pattern
  • One assert per test (when possible)
  • Descriptive test names
  • Group tests by functionality

üéØ Code Coverage

  • Aim for 80%+ coverage of important code
  • Test edge cases and error handling
  • Cover all conditional logic branches
  • Use --coverage flag

üîß Test Utilities

  • Create factory functions for test data
  • Use beforeEach/afterEach for cleanup
  • Parameterize tests for different inputs
  • Create helper functions for common operations

‚ú® Unit Testing Best Practices

  • Tests should be fast and independent
  • Test behavior, not implementation
  • Use clear names for tests and variables
  • Refactor tests alongside code regularly
  • Write tests before or with code (TDD)
